# Log Monitoring & Analysis

### Objective
The goal of this task is to understand, monitor, and analyze system logs to detect security incidents such as failed login attempts, anomalies, and suspicious behavior.

---

## Tools Used
- Linux System Logs
- Manual Log Analysis using terminal commands
- SIEM Concepts (Basic understanding)
- Optional Tool: Splunk Free (conceptual)

---

## Log Types Analyzed
- Authentication Logs (`auth.log`)
- System Logs (`syslog`)
- Security-related Events

---

## Key Activities Performed
1. Identified different log types
2. Analyzed authentication logs
3. Detected failed login attempts
4. Identified anomalies in logs
5. Correlated multiple log events
6. Learned SIEM fundamentals
7. Created basic alert rules
8. Documented findings in a report

---

## Findings Summary
- Multiple failed SSH login attempts detected
- Suspicious login attempts from unknown IPs
- Repeated authentication failures indicating possible brute-force attack
- Correlation between failed logins and system warnings

---

## Outcome
This task helped build foundational incident detection and log analysis skills essential for cybersecurity operations.
